# Инструкция по запуску сервисов и наполнению баз

## Вариант 1 — Docker Compose (рекомендуется)

Требуется установленный Docker с плагином Compose.

```bash
# распаковать проект и перейти в каталог
tar -xzf kanban.tar.gz
cd kanban

# собрать и запустить все сервисы
docker compose up --build
```

После старта поднимаются три контейнера: `db` (PostgreSQL 16), `backend` (FastAPI),
`frontend` (React + nginx).

**Доступ:**
- Приложение — http://localhost:3000
- Swagger (документация API) — http://localhost:8000/docs
- Демо-пользователи: `alice` (owner), `bob` (writer), `carol` (reader) — пароль у всех `password123`

**Наполнение базы происходит автоматически** при первом старте backend:
контейнер выполняет `alembic upgrade head` (применение миграций), затем
`python -m scripts.seed` (демо-данные). Скрипт seed идемпотентен — повторный запуск
не дублирует данные.

### Установка Docker (Ubuntu), если не установлен

```bash
sudo apt update
sudo apt install docker.io docker-compose-v2
sudo systemctl enable --now docker
sudo usermod -aG docker $USER
# перелогиниться (выйти/войти) — членство в группе docker подхватится
```

## Вариант 2 — локальный запуск без Docker

Подходит для разработки; база может быть на SQLite (PostgreSQL не обязателен).

### Backend

```bash
cd kanban/backend
pip install -r requirements.txt --break-system-packages

export DATABASE_URL='sqlite:///./dev.db'
export JWT_SECRET='dev_secret'

python -m alembic upgrade head     # создание схемы (миграции)
python -m scripts.seed             # наполнение демо-данными
uvicorn app.main:app --reload --port 8000
```

Backend доступен на http://localhost:8000, Swagger — http://localhost:8000/docs.

### Frontend (в отдельном терминале)

```bash
cd kanban/frontend
npm install
npm run dev
```

Vite поднимет dev-сервер (по умолчанию http://localhost:5173) и проксирует `/api` на :8000.

## Наполнение базы вручную

```bash
cd kanban/backend
python -m scripts.seed
```

Создаёт трёх пользователей, одну доску с колонками «Очередь / В работе / Готово»,
несколько карточек, назначенных исполнителей и комментарий.

## Проверка работоспособности (smoke-тест API)

```bash
# получить токен
TOKEN=$(curl -s -X POST localhost:8000/api/auth/login \
  -H 'Content-Type: application/json' \
  -d '{"login":"alice","password":"password123"}' | python3 -c "import sys,json;print(json.load(sys.stdin)['auth_token'])")

# список досок
curl -s localhost:8000/api/boards -H "Authorization: Bearer $TOKEN"
```

## Переменные окружения

| Переменная | Назначение | Значение по умолчанию |
|---|---|---|
| `DATABASE_URL` | строка подключения к БД | postgresql+psycopg2://kanban:kanban@db:5432/kanban |
| `JWT_SECRET` | секрет подписи JWT | dev_secret_change_me |
| `JWT_TTL_HOURS` | срок жизни токена (часы) | 24 |

Для продакшена обязательно задать длинный случайный `JWT_SECRET`.
